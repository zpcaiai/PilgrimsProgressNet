# 章节提升与创意评审（玩法侧）

对 16 个章节实现与核心系统做了一次通读评审。结论先行:**叙事与"灵性状态模型"是这游戏的宝石,
关卡机制则普遍"浅一层"**——每章往往只有 1–2 个真正的交互点,大量位移是被动的,且很多机制对玩家
"不可见"(只有盯着血条才知道在变)。下面按"全局高杠杆 → 逐章亮点 → 好创意 → 准确性勘误 → 路线图"组织。

---

## 一、全局最高杠杆(改一处,全局受益)

1. **让"看不见的状态"可见可感。** HUD 只显示 Faith/Hope/Despair 三条,但实际有 12 个状态影响玩法:
   `weariness` 拖慢移动、`shame` 影响对话门槛、`watchfulness` 决定迷魂地——全都没有任何提示。
   建议:HUD 增加"负重/警醒"小指示;泥潭/绝望上升时配**音效 + 暗色粒子 + 一次性 toast**,
   让玩家知道"此刻、因何"在掉状态。这是性价比最高的一步。

2. **激活已写好但从未调用的"诱惑抵抗"系统。** `SpiritualStateManager.get_temptation_resistance()` /
   `can_resist_temptation()` 已实现,但**全代码没有任何地方调用**(已核实)。当前对话选项只用
   `faith_min: 50` 这种"裸阈值"门控。把它接进对话条件(`"temptation": "comfort_shortcut"`),
   就能把"够不够 50 信心"升级成"用 辨识+毅力 对抗 疲惫"的**道德博弈**——同一段对话会因你此刻的
   灵性姿态而呈现不同选项。这是把现有系统潜力释放出来,而非加新功能。

3. **对话选项加"后果预览"。** 选项现在是纯文本按钮;鼠标悬停/选中时显示
   `[+谦卑8 −骄傲4 · 设置:accepted_help]`,正向绿、负向红、旗标蓝。让玩家理解这套"灵性经济",
   每个选择都更有分量。

4. **NPC 应"读"玩家状态并回应。** 对话目前是无状态的:不记得你来过、不看你此刻 shame/fear/pride。
   给关键 NPC(Apollyon、牧人、Hopeful)加状态分支——"我看见你眼里的羞愧"(shame>60)、
   "别让骄傲替你说话"(pride 高)。高冲击的反馈,改动集中在对话 JSON。

5. **同伴 Hopeful 升级为"有来有往"。** 现在只是 7 秒说一句、迷魂地帮你减困意,且只有 4 句循环、
   很快重复。给它:按章节/玩家状态变化的台词、玩家可回应的选项(一起祷告/求建议)、简单寻路。
   让"结伴"成为关系而非脚本。

6. **机制"组合 + 反向"打破后期重复。** 读完 16 章,危险类型在第 8 章前就见全了,后段缺新鲜感。
   两个低成本方向:**叠加危险**(泥潭+假地、箭雨+泥潭同时)、**反向恩典区**(跪下/认罪才回复信望的花园),
   让"恩典也是主动的"在机制上成立。

## 三、几个我特别喜欢的"好创意"

- **重担携带你的具体罪。** 落入坟墓的重担网格按你前几章的选择**着色**(欺哄=紫、骄傲=红),
  在坟里被恩典"转化"褪色,之后这色调从玩家自身光环淡出。把"赦免"做成可见的转化。
- **控诉卡牌(屈辱谷)。** Apollyon 控诉阶段,谎言以卡片飘向你,可**逐条用对话反驳**,比通用动作更减"影响力"。
- **见证之云(美宫/释经者)。** 墙上挂历代朝圣者画像,走近时微亮并低语其名;读一堂课触发一段**别人活出该美德**的闪回。把"群羊见证"做成空间叙事。
- **牢房的记忆(怀疑堡)。** 牢里浮现你过往失败的影子(早期敌人、Apollyon 的话),走过它们并"想起恩典都赎回了它们"才找到应许钥匙。
- **圣城的镜子。** 门开后里面是一面镜,你看见自己的脸,旁白:"你不是靠力量到达,而是被恩典держ着。每一步都有人托住。如今,安息。"

## 四、准确性勘误(评审里几处"未实现"其实已实现)

- **崩溃/悔改流程已实现**:在 `scripts/core/Main.gd`(`_on_collapse`,绝望≥100 弹认罪面板,
  `repentance_started/completed`),不在灵性管理器里——所以"没有处理器"的说法不成立。
- **暂停菜单已实现**:`Main.gd._open_pause`(Esc)。
- **CombatHUD 脚本存在**:`scripts/ui/CombatHUD.gd`(2.2KB)——它是"战斗中未充分可见/未被各章显示",
  而非"没有脚本"。屈辱谷里把它显示出来即可。
- **确属未用**:`get_temptation_resistance()` / `can_resist_temptation()` 全代码无调用(已核实),是真正的"待接线系统"。
- 章节 JSON 里的 `spiritual_theme`、`core_mechanic` 字段**从不被读取/展示**——可用于章节开场旁白或教学 toast。
- 路线变体(MVP/竖切)在 `ChapterManager` 里存在但**未暴露到标题界面**,属"死代码"。

## 五、建议优先级(从高到低)

1. **可见性与反馈**:HUD 全状态/负重指示、危险的音效+粒子+toast、对话后果预览、CombatHUD 在屈辱谷显示。
2. **释放已有系统**:接入诱惑抵抗、NPC 读状态分支、`spiritual_theme/core_mechanic` 用于开场教学。
3. **关卡深度**:逐章"单点最高价值"那列;叠加/反向危险;凉亭/牧人/Hopeful 的关系化。
4. **收束与重玩**:圣城欢迎场景与反思;章节选择 / NG+ / 对话快进;暴露路线变体。
5. **打磨与无障碍**:空间音频、状态驱动的音乐分层、色盲/重映射按键、危险参数化进章节 JSON。

> 架构本身是优点(纯代码、零编辑器连线、JSON 数据层),不要为加内容而引入预制体/编辑器连线。
> 真正缺的不是更多功能,而是**已有功能的深度**:让每个灵性状态在世界里都有视觉/听觉/机制的显化。
